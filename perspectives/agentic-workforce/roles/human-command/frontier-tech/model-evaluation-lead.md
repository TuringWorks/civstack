# Model Evaluation Lead

## Mission

Own the evaluation system that determines whether AI models are capable, safe, robust, fair, secure, efficient, and ready for deployment.

## When Needed

- AI systems are released to users, workers, citizens, developers, or infrastructure.
- Models operate in regulated, safety-critical, adversarial, or rights-impacting contexts.
- Capability claims need proof.

## Core Jobs To Be Done

- When model performance is claimed, test it against realistic and adversarial tasks.
- When models are updated, detect regressions before users are affected.
- When risk is high, define gates and human-review thresholds.
- When failures occur, convert them into evals and mitigations.

## Responsibilities

- Maintain eval strategy, benchmark integrity, red-team process, safety rubrics, production monitoring, and release recommendations.
- Coordinate product, research, governance, legal, security, and domain experts.

## AI Delegation

- Evaluation agent, red-team prompt generator, trace analyzer, rubric scorer, benchmark contamination checker, failure clustering agent.

## Robot Delegation

- For embodied AI, robot test fleet executes physical-world scenarios under safety controls.

## Human Boundary

Evaluation thresholds, release recommendations, risk acceptance, benchmark interpretation, and public claims remain human-led.

## Metrics

- Eval coverage.
- Regression detection.
- Red-team finding severity.
- Production failure rate.
- Bias/robustness scores.
- Time from failure to eval update.
- Benchmark integrity.

## Context Needed By Agents

Use cases, risk tiers, eval datasets, rubrics, model versions, prompts, production traces, incident records, policy requirements, and user impact data.
