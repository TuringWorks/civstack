# Chip Architecture Lead

## Mission

Design processor, accelerator, memory, interconnect, and system architectures that serve national AI, defense, telecom, industrial, and scientific workloads.

## When Needed

- Strategic workloads need performance, efficiency, security, or supply assurance beyond commodity chips.
- New AI, robotics, sensor, or edge workloads require specialized silicon.
- Design decisions must coordinate with fabs, packaging, software, and supply chain.

## Core Jobs To Be Done

- When workloads evolve, define chip architecture so compute capability matches real use.
- When fabrication options are constrained, adapt architecture to available processes and packaging.
- When security matters, design trust, isolation, and provenance into hardware.
- When software must exploit hardware, coordinate compilers, kernels, and developer tooling.

## Responsibilities

- Own architecture roadmap, workload analysis, performance modeling, design tradeoffs, security features, and hardware/software co-design.

## AI Delegation

- Workload profiling agent, architecture simulation agent, EDA copilot, design-rule checker, verification assistant, documentation agent.

## Robot Delegation

- Low direct fit; robot roles support labs, prototyping, and test hardware logistics.

## Human Boundary

Architecture tradeoffs, security commitments, tapeout readiness, customer promises, and export-control implications remain human-led.

## Metrics

- Performance per watt.
- Workload fit.
- Verification coverage.
- Tapeout success.
- Software ecosystem readiness.
- Security findings.
- Strategic dependency reduction.

## Context Needed By Agents

Target workloads, process constraints, packaging options, EDA flows, security requirements, export controls, software stack, and customer roadmap.
