# AI Security Lead

## Mission

Protect AI systems, data, models, tools, users, and infrastructure from misuse, compromise, exfiltration, manipulation, and supply-chain risk.

## When Needed

- AI systems access sensitive data, tools, code, infrastructure, customer accounts, or public channels.
- Models can be attacked through prompts, data poisoning, model extraction, tool misuse, or supply-chain compromise.
- AI systems are deployed in adversarial contexts.

## Core Jobs To Be Done

- When AI systems are designed, threat-model model, data, prompt, tool, and deployment surfaces.
- When agents use tools, constrain permissions so compromise cannot cascade.
- When attacks occur, detect, contain, investigate, and harden.
- When vendors or models are integrated, assess supply-chain and model provenance risk.

## Responsibilities

- Own AI threat models, security evals, access controls, sandboxing, logging, incident playbooks, vendor security, and misuse monitoring.

## AI Delegation

- Threat modeling agent, prompt injection tester, code/security scanner, incident triage agent, dependency risk agent, log analyzer.

## Robot Delegation

- Physical data center/security inspection support.

## Human Boundary

Security architecture, incident command, disclosure, vendor acceptance, and high-risk permission grants remain human-led.

## Metrics

- Security eval coverage.
- Prompt/tool attack success rate.
- Mean time to detect/respond.
- Privilege minimization.
- Vendor risk closure.
- Data exfiltration incidents.
- Misuse escalations.

## Context Needed By Agents

Architecture, tool permissions, data classifications, threat models, logs, vulnerabilities, vendor assessments, incident history, and red-team results.
