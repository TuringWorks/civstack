# Model Training Lead

## Mission

Own the technical execution of model training runs: data mixture, architecture, optimization, infrastructure coordination, reproducibility, and training-run learning.

## When Needed

- Training runs are expensive, long-running, or technically complex.
- Models require coordinated data, architecture, infrastructure, and evaluation choices.
- Failed runs create major cost, schedule, or strategic risk.

## Core Jobs To Be Done

- When a model objective is defined, turn it into a reproducible training plan.
- When training begins, monitor signals so failures are caught early.
- When results arrive, diagnose whether gains came from data, architecture, compute, or evaluation artifacts.
- When scaling decisions arise, recommend next runs based on evidence.

## Responsibilities

- Define training configs, dataset mixtures, token budgets, compute plan, checkpoint cadence, observability, and abort criteria.
- Coordinate data engineering, platform, research, evaluation, security, and governance.

## AI Delegation

- Training run monitor, config comparison agent, data mixture analyst, scaling-law analyst, failure diagnosis agent, experiment documentation agent.

## Robot Delegation

- Data center hardware logistics and inspection support through robot personnel.

## Human Boundary

Launch/abort decisions for major runs, sensitive data use, architecture strategy, risk acceptance, and post-training interpretation remain human-owned.

## Metrics

- Training stability.
- Compute utilization.
- Cost per useful capability gain.
- Reproducibility.
- Failed-run rate.
- Documentation completeness.
- Time to diagnose failures.

## Context Needed By Agents

Training configs, dataset lineage, cluster telemetry, optimizer settings, model architecture, checkpoint metrics, eval snapshots, incident logs, and budget constraints.
