# AI Platform Lead

## Mission

Provide reliable, secure, cost-efficient infrastructure for training, evaluation, deployment, monitoring, and continuous improvement of AI systems.

## When Needed

- Multiple teams build or operate AI systems.
- Training and inference costs, reliability, and security must be managed centrally.
- Models require standardized deployment, eval, logging, and rollback.

## Core Jobs To Be Done

- When teams need AI capability, provide reusable infrastructure so each team does not rebuild the stack.
- When models are deployed, make observability, security, scaling, and rollback default.
- When costs rise, optimize compute, caching, routing, batching, and model selection.
- When incidents happen, support rapid diagnosis and recovery.

## Responsibilities

- Own model registry, serving platform, eval pipelines, prompt/config management, telemetry, access control, cost controls, and developer experience.

## AI Delegation

- Capacity optimizer, deployment assistant, incident copilot, cost anomaly detector, documentation agent, platform support agent.

## Robot Delegation

- Data center inspection and hardware logistics where physical infrastructure is owned.

## Human Boundary

Platform architecture, access policy, production release controls, major vendor decisions, and risk acceptance remain human-led.

## Metrics

- Platform uptime.
- Deployment lead time.
- Inference latency/cost.
- Model rollback time.
- Developer satisfaction.
- Security findings.
- Reuse rate across teams.

## Context Needed By Agents

Platform architecture, model registry, deployment configs, observability data, cost data, access rules, incident logs, and service ownership.
