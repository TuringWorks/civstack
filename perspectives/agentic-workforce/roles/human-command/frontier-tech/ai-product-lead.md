# AI Product Lead

## Mission

Turn AI capability into useful, trusted, economically viable products and workflows that solve real user problems.

## When Needed

- AI models or agents must be embedded in products, services, operations, or public systems.
- User trust, workflow fit, reliability, and adoption matter.
- AI capability must be translated into measurable value.

## Core Jobs To Be Done

- When AI can do many things, choose product jobs where capability creates durable value.
- When users interact with AI, design workflows that make uncertainty, review, and correction natural.
- When adoption lags, identify whether the problem is trust, quality, latency, cost, UX, or incentives.
- When AI changes work, redesign human roles and escalation paths.

## Responsibilities

- Own user research, product requirements, workflow design, launch plan, adoption metrics, feedback loops, and product safety constraints.

## AI Delegation

- User research summarizer, prototype generator, support ticket analyst, experiment analyst, prompt testing agent, documentation agent.

## Robot Delegation

- For robot products, coordinate physical-world task design and user acceptance testing.

## Human Boundary

Product strategy, user trust, pricing, launch decision, failure disclosure, and rights-impacting workflow design remain human-owned.

## Metrics

- Task success.
- Adoption.
- Retention.
- User trust.
- Error/correction rate.
- Time saved.
- Cost per successful task.
- Escalation quality.

## Context Needed By Agents

User personas, workflows, product requirements, model limits, eval results, telemetry, support data, legal/privacy constraints, and competitive alternatives.
