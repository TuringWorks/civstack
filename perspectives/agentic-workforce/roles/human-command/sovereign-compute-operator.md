# Sovereign Compute Operator

## Mission

Operate nationally strategic compute capacity for AI, science, defense, public services, and industry with resilience, efficiency, security, and fair allocation.

## When Needed

- Frontier AI or science requires large compute clusters.
- Compute supply is constrained by chips, power, cooling, facilities, or geopolitics.
- Sensitive workloads require domestic or trusted infrastructure.

## Core Jobs To Be Done

- When demand exceeds capacity, allocate compute so national priorities advance.
- When clusters are expensive, optimize utilization, energy, cooling, scheduling, and reliability.
- When workloads are sensitive, enforce security, data isolation, and auditability.
- When hardware evolves, plan procurement and lifecycle replacement.

## Responsibilities

- Manage capacity planning, workload scheduling, energy contracts, data center operations, security controls, and user onboarding.
- Coordinate AI labs, universities, cloud providers, utilities, chip suppliers, and security teams.

## AI Delegation

- Capacity planner, workload optimizer, cost forecaster, energy scheduler, security monitoring agent, procurement analyst.

## Robot Delegation

- Data center inspection, cable/hardware handling, rack support, inventory, and physical security patrol support.

## Human Boundary

Strategic allocation, security exceptions, procurement commitments, classified/sensitive workload policy, and emergency prioritization remain human-accountable.

## Metrics

- Utilization.
- Job wait time.
- Cost per training/inference unit.
- Energy efficiency.
- Uptime.
- Security incidents.
- Priority mission throughput.

## Context Needed By Agents

Cluster inventory, job queues, user priorities, power/cooling capacity, security policies, procurement pipeline, model training plans, and incident history.
