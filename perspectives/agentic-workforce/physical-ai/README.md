# Non-Humanoid Physical AI Library

Purpose: define where physical AI in non-humanoid form can do work: self-driving vehicles, autonomous farm equipment, harvesters, loaders, drones, marine systems, rail systems, warehouse AMRs, construction/mining machines, and inspection platforms.

Use this library when designing autonomous machine workforces where the machine shape is optimized for the job rather than human-like movement.

## Folder Map

- `skills/`: platform-specific skill packs.
- `catalogs/`: shared taxonomies and safety boundaries.
- `operating-models/`: coordination models for autonomous machine fleets.
- `roles/`: human command roles, AI-personnel roles, and autonomous machine role cards.
- `checklists/`: deployment readiness and safety reviews.

## Core Principle

Use non-humanoid physical AI when the environment, task, economics, and safety case favor specialized mobility or tooling: wheels, tracks, rotors, implements, loaders, sprayers, harvest heads, forklifts, vessels, locomotives, or fixed industrial cells.

## Common Pattern

Every autonomous machine role needs:

- Human accountable owner.
- Operating design domain.
- Map/geofence or route authority.
- Sensor and perception requirements.
- Remote operations plan.
- Emergency stop and fallback behavior.
- Maintenance and inspection routine.
- Cybersecurity controls.
- Incident logging and investigation.
- Public, worker, animal, crop, property, and environmental safety plan.
